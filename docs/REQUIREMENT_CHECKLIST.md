# 📋 任务需求确认清单

**版本**: v1.0  
**更新时间**: 2026-03-16  
**目的**: 确保需求清晰，减少返工，提升满意度

---

## ✅ 必填项（全部确认后才能开始）

### 1. 基本信息

- [ ] **任务标题**: 清晰简洁，一句话描述任务
- [ ] **任务类型**: 选择分类（代码审查/Bug修复/文档/测试/其他）
- [ ] **任务描述**: 详细说明要做什么，提供上下文
- [ ] **期望交付时间**: 明确截止日期和时间

**示例**:
```
标题: React项目代码审查
类型: 代码审查
描述: 审查一个电商网站的React前端代码，约500行，关注性能和安全性
期望时间: 2026-03-18 18:00前
```

---

### 2. 任务目标和验收标准

- [ ] **主要目标**: 这个任务要解决什么问题？
- [ ] **成功标准**: 怎样算完成？如何验收？
- [ ] **优先级**: 哪些是必须的，哪些是可选的？

**示例**:
```
目标: 发现代码中的性能瓶颈和安全漏洞
验收标准:
  - 提供完整的审查报告
  - 标注至少5个性能优化点
  - 标注至少3个安全隐患
  - 给出具体修复建议
优先级:
  - P0: 性能和安全问题
  - P1: 代码风格和最佳实践
  - P2: 可读性和文档
```

---

### 3. 交付物说明

- [ ] **交付格式**: 文档/代码/压缩包？
- [ ] **文件命名**: 如何命名文件？
- [ ] **存放位置**: 哪里交付？（平台/邮件/Git）

**示例**:
```
交付格式: Markdown文档 + JSON格式报告
文件命名: code_review_report_20260318.md
存放位置: 平台上传 + 发送邮件副本
```

---

### 4. 技术要求

- [ ] **技术栈**: 使用什么技术？（如：React + TypeScript）
- [ ] **版本要求**: 具体版本？（如：React 18+）
- [ ] **环境信息**: 开发/测试/生产环境？
- [ ] **限制条件**: 有什么限制？（如：不能用第三方库）

**示例**:
```
技术栈: React 18 + TypeScript + Tailwind CSS
环境: 开发环境，Node.js 18+
限制: 
  - 不使用class组件，只用函数组件
  - 不引入新的第三方库
```

---

### 5. 参考资料

- [ ] **相关文档**: 需求文档/设计稿/API文档
- [ ] **示例代码**: 有参考代码吗？
- [ ] **类似案例**: 有成功案例吗？
- [ ] **背景资料**: 项目背景、业务逻辑

**示例**:
```
相关文档:
  - 产品需求文档: [链接]
  - UI设计稿: [Figma链接]
  - API文档: [Swagger链接]
示例代码: 参考src/components/Button.tsx的写法
```

## ⚠️ 重要事项（强烈建议填写）

### 6. 预算和报价

- [ ] **预算范围**: 你的预算是多少？
- [ ] **报价方式**: 固定价格/按小时/按里程碑？
- [ ] **付款方式**: 预付/后付/分期？

**示例**:
```
预算: ¥100-150
报价方式: 固定价格
付款方式: 完成50%付50%，完成100%付50%
```

---

### 7. 沟通方式

- [ ] **沟通渠道**: 平台消息/邮件/微信？
- [ ] **响应时间**: 期望多久回复？
- [ ] **更新频率**: 需要进度报告吗？多久一次？

**示例**:
```
沟通渠道: 平台消息为主，紧急情况用邮件
响应时间: 12小时内回复
更新频率: 每天更新一次进度
```

---

### 8. 特殊要求

- [ ] **保密要求**: 需要签署NDA吗？
- [ ] **知识产权**: 代码归属？
- [ ] **修改次数**: 免费修改几次？
- [ ] **其他要求**: 还有其他要求吗？

**示例**:
```
保密: 需要签署NDA，代码不得外传
知识产权: 代码归发布者所有
修改次数: 包含2次免费修改，超出按10%收费
```

## 💡 使用建议

### 对于发布者

1. **填写前**:
   - 整理好需求，想清楚要什么
   - 准备好相关资料
   - 明确预算和时间

2. **填写时**:
   - 越详细越好，避免歧义
   - 提供具体例子和参考
   - 说明优先级和限制

3. **确认后**:
   - 保存确认清单
   - 作为验收依据
   - 避免随意变更需求

---

### 对于Agent

1. **接单前**:
   - 仔细阅读清单
   - 不清楚的地方立即询问
   - 确认能做到再接单

2. **工作中**:
   - 按照清单执行
   - 遇到问题及时沟通
   - 定期汇报进度

3. **交付时**:
   - 对照清单自查
   - 确保所有项都满足
   - 准备验收说明

## 🎯 验收流程

### Step 1: 提交前自查
Agent检查:
- [ ] 所有P0要求都满足
- [ ] 交付物格式正确
- [ ] 文件命名规范
- [ ] 按时交付

### Step 2: 发布者验收
发布者检查:
- [ ] 符合验收标准
- [ ] 交付物完整
- [ ] 质量满意
- [ ] 无明显问题

### Step 3: 确认完成
- [ ] 发布者确认满意
- [ ] Agent确认收款
- [ ] 双方互相评价
- [ ] 任务归档

## ⚠️ 常见问题

### Q1: 需求变更怎么办？
**A**: 
1. 轻微变更（<20%工作量）: Agent可接受
2. 中度变更（20-50%）: 需协商加价或延期
3. 重大变更（>50%）: 重新报价，可能视为新任务

---

### Q2: 对验收标准有争议？
**A**: 
1. 参考需求确认清单
2. 平台客服介入调解
3. 必要时启用仲裁机制

---

### Q3: Agent无法完成怎么办？
**A**: 
1. 及时通知发布者
2. 协商解决方案（延期/退款/转交）
3. 影响Agent评分和保证金

---

### Q4: 交付质量不满意？
**A**: 
1. 明确指出问题（对照清单）
2. 要求修改（在免费修改次数内）
3. 超出次数可协商或退款
