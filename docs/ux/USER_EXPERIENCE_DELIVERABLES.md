## 🎓 关键设计决策

### 1. Token 存储策略
**决策**: 使用 localStorage（记住我）和 sessionStorage（不记住）
**理由**: 
- 灵活支持"记住我"功能
- 避免每次刷新都需要重新登录
- 安全性：sessionStorage 关闭浏览器自动清除

---

### 2. 路由守卫实现
**决策**: 使用 Next.js Middleware + React AuthGuard 组件
**理由**:
- Middleware 在服务端拦截，更安全
- AuthGuard 在客户端保护，用户体验更好
- 双重保护，确保安全性

---

### 3. 欢迎引导流程
**决策**: 3步引导（角色选择 → 技能选择 → 开始使用）
**理由**:
- 简洁明了，不会让用户感到繁琐
- 收集关键信息（角色、技能）用于个性化推荐
- 可跳过，不影响用户快速上手

---

### 4. 角色系统
**决策**: 单一角色（Agent 或 发布者）
**理由**:
- 简化系统逻辑
- 避免角色混淆
- 后续可扩展为多角色

### 3. Token 过期
**问题**: Token 有效期多久？是否需要自动刷新？
**建议**: 
- 有效期：7天
- 自动刷新：是，在 Token 即将过期时（剩余1天）自动刷新
