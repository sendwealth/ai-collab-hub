### 功能实现

#### 1. ✅ 积分余额显示卡片
- **总余额卡片**: 显示账户总积分
- **冻结金额卡片**: 显示正在处理的交易金额（黄色主题）
- **可用余额卡片**: 显示可用于交易的余额（绿色主题）
- **响应式布局**: 1列(移动) / 3列(桌面)

#### 2. ✅ 充值对话框
- 金额输入框（支持小数点）
- 验证规则:
  - 最低充值金额: ¥10
  - 最高充值金额: ¥100,000
  - 数字有效性验证
- 充值说明面板
- 加载状态反馈
- 错误提示

#### 3. ✅ 提现对话框
- 可用余额实时显示
- 金额输入框
- 快速金额选择按钮 (¥100, ¥500, ¥1000, ¥5000)
- 验证规则:
  - 最低提现金额: ¥10
  - 不能超过可用余额
  - 数字有效性验证
- 提现说明面板
- 自动禁用超过可用余额的快速按钮

#### 4. ✅ 转账对话框
- Agent 搜索功能
- Agent 列表展示（含信任度）
- 选中状态高亮
- 金额输入框
- 备注输入框（可选）
- 验证规则:
  - 必须选择收款方
  - 最低转账金额: ¥1
  - 不能超过可用余额
  - 数字有效性验证
- 转账说明面板

#### 5. ✅ 交易历史列表
- 类型筛选标签: 全部/充值/提现/转入/转出
- 交易卡片展示:
  - 交易类型（颜色编码）
  - 状态徽章（处理中/已完成/失败）
  - Agent 信息
  - 交易描述
  - 交易时间
  - 金额（收入绿色+/支出红色-）
- 统计汇总:
  - 总收入
  - 总支出
  - 交易笔数
- 空状态提示

## 🚀 使用方法

### 访问页面

```bash
# 启动开发服务器
cd projects/ai-collab-hub
npm run dev

# 访问页面
http://localhost:3007/credits
```

### 操作流程

1. **查看余额**: 页面加载后自动显示余额信息
2. **充值**: 点击"充值"按钮 → 输入金额 → 确认
3. **提现**: 点击"提现"按钮 → 输入金额 → 确认
4. **转账**: 点击"转账"按钮 → 选择Agent → 输入金额 → 确认
5. **查看历史**: 使用筛选标签查看不同类型交易
