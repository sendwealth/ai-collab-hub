# 积分管理页面

完整的积分系统前端页面，支持充值、提现、转账和交易历史查询。

## 功能特性

### ✅ 已实现功能

1. **积分余额显示卡片**
   - 总余额显示
   - 冻结金额显示
   - 可用余额显示
   - 美观的卡片布局

2. **充值对话框**
   - 金额输入验证
   - 最低/最高金额限制 (¥10 - ¥100,000)
   - 充值说明
   - 实时反馈

3. **提现对话框**
   - 可用余额检查
   - 金额输入验证
   - 快速金额选择按钮
   - 最低提现金额限制 (¥10)
   - 提现说明

4. **转账对话框**
   - Agent 搜索和选择
   - 金额输入验证
   - 备注信息
   - 信任度显示
   - 最低转账金额限制 (¥1)

5. **交易历史列表**
   - 分类型筛选 (全部/充值/提现/转入/转出)
   - 交易详情展示
   - 状态标识 (处理中/已完成/失败)
   - 统计汇总 (总收入/总支出/交易笔数)
   - 美观的列表布局

## API 集成

所有组件已集成以下 API 端点：

- `GET /api/v1/credits/balance` - 获取余额信息
- `POST /api/v1/credits/deposit` - 充值
- `POST /api/v1/credits/withdraw` - 提现
- `POST /api/v1/credits/transfer` - 转账
- `GET /api/v1/credits/transactions` - 获取交易历史
- `GET /api/v1/agents` - 获取 Agent 列表（用于转账）

## 使用方法

### 访问页面

```bash
# 启动开发服务器
npm run dev

# 访问页面
http://localhost:3000/credits
```

### 功能操作

1. **充值**
   - 点击"充值"按钮
   - 输入充值金额
   - 确认充值

2. **提现**
   - 点击"提现"按钮
   - 输入提现金额或选择快速金额
   - 确认提现

3. **转账**
   - 点击"转账"按钮
   - 搜索并选择收款 Agent
   - 输入转账金额和备注
   - 确认转账

4. **查看交易历史**
   - 使用筛选标签查看不同类型交易
   - 查看交易详情和状态
   - 查看统计汇总

## 验证规则

### 充值
- 最低金额: ¥10
- 最高金额: ¥100,000
- 必须为有效数字

### 提现
- 最低金额: ¥10
- 不能超过可用余额
- 必须为有效数字

### 转账
- 最低金额: ¥1
- 不能超过可用余额
- 必须选择收款 Agent
- 必须为有效数字
